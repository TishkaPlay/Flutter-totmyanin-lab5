## 📚 Что изучили в этой работе

1. Future<T> — объект-«обещание», который вернёт результат типа T в будущем; позволяет выполнять длительные операции без блокировки UI.

2. async/await — синтаксический сахар для работы с асинхронным кодом: делает код последовательным и читаемым, при этом не блокирует основной поток.

3. Обработка ошибок — использование try/catch для перехвата исключений при сетевых запросах и парсинге JSON.

4. HTTP-запросы во Flutter — отправка GET-запросов через пакет http, парсинг ответа с помощью jsonDecode.

5. Управление состоянием — использование setState() для обновления UI после получения данных (индикатор загрузки → картинка / ошибка).

## ❓ Ответы на вопросы:

1. Что такое Future<T>? Чем отличается от обычного возвращаемого значения?
Future<T> — это объект-«обещание», который вернёт результат (типа T) не сразу, а в будущем. В отличие от обычного значения, которое возвращается мгновенно, Future позволяет выполнять длительные операции (например, запрос к серверу), не блокируя интерфейс.

2. Что делает await? Блокирует ли он весь поток выполнения?
await приостанавливает выполнение текущей асинхронной функции до тех пор, пока Future не завершится. Нет, он не блокирует весь поток — приложение остаётся отзывчивым, интерфейс не зависает, пока ждём ответ.

3. Зачем setState() вызывается дважды в _fetchPhoto()?
Первый раз (в начале): чтобы показать индикатор загрузки (_isLoading = true) перед отправкой запроса.
Второй раз (в конце): чтобы скрыть индикатор и обновить экран полученной картинкой или ошибкой (_isLoading = false).

4. Почему кнопке передаётся _fetchPhoto без скобок, а не _fetchPhoto()?
Потому что без скобок (_fetchPhoto) мы передаём ссылку на функцию — она будет вызвана только когда пользователь нажмёт кнопку. Если написать со скобками (_fetchPhoto()), функция вызовется сразу при загрузке экрана, а не по клику.

5. Чем Image.network() отличается от Image.asset()?
Image.network() загружает картинку из интернета по ссылке (URL), а Image.asset() отображает файл, который хранится локально в папке проекта (assets).
